# 노션 트러블슈팅 검색기

노션에 저장된 트러블슈팅 DB를 벡터 DB로 변환하여 의미 기반 검색이 가능하도록 구성한 RAG 시스템

## 프로젝트 개요

기존 노션 검색의 한계인 제목 기반 검색을 벗어나, 문서 내용까지 의미적으로 검색할 수 있는 구조 구현

단순 키워드 일치 여부가 아니라, 질문과 문서의 의미적 유사도를 기준으로 관련 사례를 찾는 방식

## 문제 인식

기존 노션 검색의 한계는 다음과 같음

```text
기존 노션 검색
→ 제목만 검색 가능
→ 상세 내용 검색 불가
→ 비슷한 의미의 문서 찾기 어려움
```

실제 트러블슈팅 문서는 제목보다 본문에 핵심 원인, 해결 과정, 에러 메시지, 예외 상황이 담기는 경우가 많음

따라서 제목 검색만으로는 필요한 사례를 놓치기 쉽고, 비슷한 문제를 다른 표현으로 기록한 문서도 찾기 어려운 구조

## 해결 방법

전체 흐름은 다음과 같음

```text
노션 DB Export (Markdown)
        ↓
문서 청킹 (RecursiveCharacterTextSplitter)
        ↓
임베딩 변환 (OpenAI Embeddings)
        ↓
벡터 DB 저장 (Chroma)
        ↓
의미 기반 검색
        ↓
관련 문서 반환
```

즉, 노션 문서를 텍스트 단위로 분해한 뒤 벡터로 변환하고, 질문과 가까운 의미를 가진 문서를 검색하는 구조

## 처리 단계 정리

### 1. 노션 DB Export

노션에 정리된 트러블슈팅 문서를 Markdown 형태로 Export

초기 단계에서는 정적인 Export 파일 기반으로 데이터 적재

### 2. 문서 청킹

문서 전체를 한 번에 임베딩하지 않고, 검색 가능한 작은 단위로 분할하는 단계

사용 방식

- `RecursiveCharacterTextSplitter`
- 문맥 손실을 줄이면서 검색 가능한 단위로 분해

목적

- 긴 문서 검색 효율 개선
- 관련 부분만 더 정확하게 검색 가능
- 벡터 검색 품질 향상

### 3. 임베딩 변환

분할된 텍스트를 의미를 담은 숫자 벡터로 변환하는 단계

사용 방식

- `OpenAI Embeddings`

목적

- 단어 일치가 아닌 의미 기반 검색 가능
- 비슷한 표현의 문제도 유사 문서로 연결 가능

### 4. 벡터 DB 저장

생성된 임베딩 벡터를 Chroma DB에 저장하는 단계

목적

- 빠른 유사도 검색
- 문서 단위 검색 결과 관리
- 검색용 로컬 데이터베이스 구성

### 5. 의미 기반 검색

사용자 질문을 동일하게 임베딩한 뒤, 저장된 문서 벡터와 비교하여 유사한 문서를 검색하는 단계

목적

- 키워드가 정확히 일치하지 않아도 관련 사례 검색
- 에러 원인, 증상, 해결 방식이 비슷한 문서 탐색 가능

### 6. 관련 문서 반환

유사도가 높은 문서를 사용자에게 반환하는 단계

반환 형태 예시

- 관련 트러블슈팅 제목
- 본문 일부
- 유사도 높은 문서 목록
- 후속 응답 생성용 컨텍스트

## 사용 기술

- Python
- LangChain
- OpenAI API
- Chroma DB

## 사용 방법

기본 사용 흐름은 다음과 같음

```text
1. 노션 DB Export (Markdown)
2. 벡터 DB 저장
3. 키워드 또는 질문 입력
4. 관련 트러블슈팅 문서 반환
```

즉, 사용자는 문제 상황을 입력하고, 시스템은 의미적으로 가까운 트러블슈팅 사례를 검색해 반환하는 방식

## 기대 효과

- 제목 중심 검색보다 더 넓은 검색 가능 범위 확보
- 비슷한 장애 사례 재활용 가능
- 반복되는 문제 해결 시간 단축
- 팀 내부 문제 해결 지식의 검색 가능성 향상

## 한계

현재 구조의 한계는 다음과 같음

- Export 기반이므로 실시간 반영 어려움
- 청킹 품질에 따라 검색 성능 차이 발생
- 문서 구조가 일정하지 않으면 검색 품질이 흔들릴 수 있음
- 단순 검색 단계에서는 최종 응답 생성 품질이 별도 보완 필요

## 개선 사항

```text
기본 청킹
→ Semantic Chunking 개선 예정

Export 방식
→ Notion API 연동 예정 (실시간 동기화)
```

추가 개선 방향은 다음과 같음

- 메타데이터 기반 필터링 추가
- 검색 결과 재정렬 방식 보완
- 검색 결과 요약 응답 생성 단계 추가
- 운영용 서비스 형태로 UI 연결

## 정리

노션 트러블슈팅 검색기는 노션 문서를 벡터 DB로 변환하여 의미 기반 검색이 가능하도록 만든 RAG 시스템

핵심은 Markdown Export 문서를 청킹하고, 임베딩으로 변환한 뒤, Chroma DB에 저장하여 질문과 유사한 문제 해결 문서를 검색하는 구조

즉, 제목 검색 중심의 한계를 넘어서 실제 문제 해결 기록을 더 잘 찾기 위한 검색 시스템
