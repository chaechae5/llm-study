# LLM (Large Language Model)

대규모 언어 모델

## 한 줄 정의

> 방대한 양의 텍스트를 학습해, 언어를 이해하고 생성하도록 만든 AI 모델

## 퍼셉트론과의 연결

LLM도 완전히 새로운 개념이라기보다, 기존 신경망 구조가 대규모로 확장된 형태

```text
퍼셉트론
→ 다층 퍼셉트론
→ 신경망
→ 딥러닝
→ 대규모 언어 모델(LLM)
```

즉, 기본 계산 원리는 신경망과 동일

다만 규모, 데이터, 파라미터 수, 학습 방식이 훨씬 커진 형태

LLM을 단순화해서 보면, 텍스트를 입력받아 다음에 올 단어 또는 토큰을 예측하는 거대한 신경망 구조

## 입력과 출력 기준 정리

| 구분 | 내용 |
| --- | --- |
| 입력 | 텍스트, 코드, 질문, 문장 |
| 출력 | 답변, 요약, 번역, 코드 생성 |
| 학습 데이터 | 인터넷 문서, 책, 논문, 코드, 각종 텍스트 |
| 핵심 목적 | 다음 토큰 예측 |
| 기반 구조 | Transformer 기반 신경망 |

## LLM의 핵심 동작 방식

LLM은 텍스트를 그대로 이해하는 것이 아니라, 텍스트를 잘게 나눈 단위인 토큰을 입력으로 사용

기본 동작 흐름은 다음과 같음

```text
텍스트 입력
→ 토크나이저로 토큰 분할
→ 토큰을 숫자로 변환
→ 임베딩으로 의미 표현
→ 신경망이 다음 토큰 예측
→ 예측 결과를 다시 텍스트로 변환
```

즉, 사람이 문장을 읽는 방식과 동일하다기보다, 문장을 숫자로 바꾸고 그 패턴을 바탕으로 다음 토큰을 예측하는 구조

## 왜 LLM이 중요한가

생성형 AI 서비스 대부분이 LLM 위에서 동작

예시

- 챗봇
- 문서 요약
- 번역
- 검색 보조
- 코드 생성
- 에이전트 시스템

현실적으로는 직접 LLM을 새로 학습시키기보다, 기존 모델을 API 형태로 가져와 서비스에 연결하는 방식이 일반적

```text
직접 모델 학습
→ 막대한 데이터, 연산 자원, 비용 필요

기존 LLM 활용
→ API 또는 오픈소스 모델 기반으로 빠르게 서비스 구현 가능
```

즉, 실무에서는 모델 자체를 만드는 것보다 이미 존재하는 LLM을 서비스 문제에 맞게 잘 활용하는 능력이 더 중요

## 개념 정리

### 1. 퍼셉트론

가장 기본적인 신경망 계산 단위

```text
입력값 × 가중치
→ 합산
→ 출력 생성
```

신경망과 LLM의 가장 기초가 되는 개념

### 2. 가중치 학습

모델이 더 나은 답을 내도록 내부 값을 조정하는 과정

예측 결과가 정답과 다르면, 오차를 줄이는 방향으로 가중치 수정

즉, 학습의 본질은 가중치 조정

### 3. 미분 / 경사하강법

오차를 줄이기 위해 가중치를 어느 방향으로 얼마나 바꿔야 하는지 찾는 방법

- 미분: 변화 방향 계산
- 경사하강법: 오차가 줄어드는 방향으로 조금씩 이동

즉, 학습을 가능하게 하는 핵심 수학적 원리

### 4. 임베딩

단어나 문장의 의미를 숫자 벡터로 바꾸는 과정

컴퓨터는 단어 자체를 이해하지 못하므로, 의미를 담은 숫자 형태로 변환 필요

핵심 특징은 다음과 같음

```text
비슷한 의미의 단어
→ 비슷한 벡터 위치
```

예시

```text
사과, 배, 포도
→ 과일이라는 공통 의미 반영

고양이, 강아지
→ 동물이라는 의미상 가까운 위치 형성
```

즉, 임베딩은 의미를 숫자 공간에 표현한 결과

### 5. 토크나이저

텍스트를 모델이 처리할 수 있는 단위로 분해하는 도구

문장을 그대로 넣는 것이 아니라, 토큰이라는 작은 단위로 나눈 뒤 숫자로 변환

예시

```text
"나는 밥을 먹었다"
→ ["나는", "밥", "을", "먹었다"] 또는 더 작은 단위로 분해
```

LLM은 이 토큰 단위 기준으로 입력과 출력을 처리

### 6. LLM

앞에서 본 모든 요소가 결합된 대규모 언어 모델

구성 요소를 단순하게 정리하면 다음과 같음

```text
토크나이저
→ 텍스트 분해

임베딩
→ 의미를 숫자로 표현

신경망
→ 패턴 학습 및 다음 토큰 예측

가중치 학습
→ 더 정확한 예측을 위한 내부 값 조정
```

즉, LLM은 텍스트를 토큰으로 바꾸고, 숫자 벡터로 표현한 뒤, 거대한 신경망을 통해 다음 토큰을 예측하는 구조

### 7. Hugging Face

오픈소스 모델과 관련 도구를 공유하는 대표적인 플랫폼

주요 역할은 다음과 같음

- 사전학습 모델 공유
- 토크나이저 제공
- 학습 및 추론 라이브러리 제공
- 모델 다운로드 및 파인튜닝 지원

즉, LLM 생태계의 모델 허브 역할

## 핵심 요약

```text
퍼셉트론
→ 신경망의 최소 계산 단위

가중치 학습
→ 오차를 줄이도록 내부 값 조정

미분 / 경사하강법
→ 가중치를 어떻게 바꿀지 결정하는 원리

토크나이저
→ 텍스트를 토큰으로 분해

임베딩
→ 단어 의미를 숫자로 표현

LLM
→ 이 모든 구조를 대규모로 결합한 언어 모델

Hugging Face
→ 학습된 모델을 공유하고 활용하는 플랫폼
```

## 정리

LLM은 텍스트를 처리하기 위해 만들어진 대규모 신경망 모델

텍스트를 토큰으로 나누고, 이를 숫자 벡터로 변환한 뒤, 방대한 가중치를 가진 신경망을 통해 다음 토큰을 예측하는 방식으로 동작

즉, LLM은 퍼셉트론, 가중치 학습, 임베딩, 토크나이저, 경사하강법 같은 개념이 결합된 결과물

실무에서는 이를 직접 처음부터 만드는 경우보다, 기존 LLM을 API 또는 오픈소스 모델 형태로 활용해 서비스에 적용하는 방식이 일반적
